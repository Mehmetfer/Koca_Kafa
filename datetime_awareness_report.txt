TimestampUtc: 2026-06-19T15:58:35Z
Live runtime at test: 19 Haziran 2026 | Cuma | 18:58

PROBLEM
-------
"Bugün günlerden hangi gün?" → model guessed Pazartesi instead of real weekday (Cuma).
Root cause: no trusted runtime date/time in system prompt; LLM hallucinated the day.

SOLUTION
--------
1. Runtime Context (Core/RuntimeContext/DateTimeContext.cs)
   - CaptureNow() reads local system clock (tr-TR culture).
   - Exposes: Current Date, Current Day, Current Time, Time Zone.
   - Unavailable() when capture fails.

2. DateTime Awareness Engine (Services/Cognitive/DateTimeAwarenessEngine.cs)
   - IsDateTimeQuestion() detects gün/tarih/saat queries.
   - BuildPromptBlock() injects RUNTIME CONTEXT into every system prompt.
   - BuildAnswerDirective() forbids guessing when user asks date/time.
   - TryBuildDirectReply() bypasses LLM with deterministic answer.
   - Fallback when context unavailable: "Bunu doğrulayamıyorum baba."

3. System Prompt (ChatApplicationService.ComposeSystemPrompt)
   - RUNTIME CONTEXT block prepended to all prompts (Current Date/Day/Time).
   - TARİH/SAAT MODU directive added for datetime questions.
   - Empathy suppressed for datetime queries (same as knowledge).

4. Direct Reply Path (ChatApplicationService.SendMessageAsync)
   - Datetime questions answered before Ollama call (no LLM guess).

5. Intent Routing (Core/IntentAnalyzer.cs)
   - Datetime questions → FactualQuery @ 0.92 confidence.

6. Empathy / Quality Guards
   - EmpathyEngine: empathy disabled for datetime questions.
   - SelfCheckEngine: no empathy-first enforcement on datetime.
   - ResponseQualityEngine: no humble fallback on datetime questions.
